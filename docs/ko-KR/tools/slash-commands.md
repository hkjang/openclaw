# 슬래시 명령어

채팅에서 `/`로 시작하는 명령어와 디렉티브를 사용하여 에이전트, 세션, Gateway를 제어할 수 있습니다.

## 명령어 vs 디렉티브

| 구분         | 명령어 (Command)               | 디렉티브 (Directive)            |
| ------------ | ------------------------------ | ------------------------------- |
| 동작         | 독립적으로 실행                | 메시지에서 제거 후 에이전트에 전달 |
| 예시         | `/status`, `/reset`, `/help`   | `/think high`, `/verbose on`    |
| 에이전트     | 에이전트 호출하지 않음         | 에이전트가 나머지 메시지 처리   |

디렉티브는 메시지와 함께 사용합니다:

```
/think high 이 코드를 분석해줘
```

→ 에이전트는 "이 코드를 분석해줘"만 받고, thinking은 `high`로 설정됩니다.

## 전체 명령어 목록

### 정보 조회

| 명령어                | 설명                       |
| --------------------- | -------------------------- |
| `/help`               | 도움말 표시                |
| `/commands`           | 사용 가능한 명령어 목록    |
| `/status`             | 현재 세션/Gateway 상태     |
| `/whoami`             | 현재 사용자 정보           |
| `/usage [mode]`       | 사용량 표시                |

**`/usage` 모드:**
- `off` — 사용량 표시 끄기
- `tokens` — 토큰 사용량만
- `full` — 전체 사용량 (비용 포함)
- `cost` — 비용 정보만

### 세션 관리

| 명령어                | 설명                        |
| --------------------- | --------------------------- |
| `/new`                | 새 세션 시작                |
| `/reset`              | 현재 세션 초기화            |
| `/compact [지침]`     | 컨텍스트 압축               |
| `/abort`              | 실행 중인 에이전트 중단     |
| `/subagents`          | 서브에이전트 목록/상태      |

### 모델 및 에이전트

| 명령어                | 설명                        |
| --------------------- | --------------------------- |
| `/model [이름]`       | 모델 전환 (피커 또는 직접)  |
| `/agent [id]`         | 에이전트 전환               |
| `/skill <이름>`       | 특정 스킬 실행              |

### 설정 변경

| 명령어                | 설명                        |
| --------------------- | --------------------------- |
| `/config show`        | 현재 설정 표시              |
| `/config get <key>`   | 특정 설정값 조회            |
| `/config set <k> <v>` | 설정값 변경 (디스크 저장)   |
| `/config unset <key>` | 설정값 제거                 |
| `/debug show`         | 런타임 오버라이드 표시      |
| `/debug set <k> <v>`  | 런타임 오버라이드 설정      |
| `/debug unset <key>`  | 런타임 오버라이드 제거      |
| `/debug reset`        | 모든 오버라이드 초기화      |

> `/config`는 **디스크에 저장**됩니다. `/debug`는 **런타임 전용** (재시작 시 사라짐)입니다.

### 큐 관리

| 명령어                | 설명                        |
| --------------------- | --------------------------- |
| `/queue <mode>`       | 큐 모드 변경                |

모드: `collect`, `steer`, `followup`, `interrupt`

## 전체 디렉티브 목록

### 사고 모드

```
/think <level>        # off, minimal, low, medium, high, xhigh
/t <level>            # 축약형
/think:<level>        # 콜론 형식
```

디렉티브만 단독으로 보내면 세션 기본값으로 설정됩니다:

```
/think medium         # 이후 모든 메시지에 medium 적용
```

### 상세 모드

```
/verbose on           # 도구 실행 메타데이터 전송
/verbose full         # 메타데이터 + 도구 출력 전송
/verbose off          # 기본값
/v on                 # 축약형
```

### 추론 표시

```
/reasoning on         # 추론 과정을 별도 메시지로 표시
/reasoning off        # 추론 숨기기
/reasoning stream     # 추론을 스트리밍으로 표시
```

### 상승 권한

```
/elevated on          # Gateway 호스트에서 실행 (승인 유지)
/elevated ask         # on과 동일
/elevated full        # Gateway 호스트 + 자동 승인
/elevated off         # 비활성화
```

## 설정

### 텍스트 명령어

```json5
{
  commands: {
    text: true,          // 채팅에서 /... 명령어 파싱 (기본: true)
  },
}
```

### 네이티브 명령어 (Discord/Telegram)

```json5
{
  commands: {
    native: "auto",      // auto | true | false
  },
}
```

- `auto`: 플랫폼이 지원하면 자동 등록
- `true`: 항상 등록
- `false`: 비활성화

Discord는 자동완성, Telegram/Slack은 버튼 메뉴를 사용합니다.

### 셸 명령어

```json5
{
  commands: {
    bash: false,         // ! <cmd> 로 호스트 셸 접근 (기본: false)
  },
}
```

활성화 시 `! ls -la` 같은 호스트 셸 명령어를 채팅에서 실행할 수 있습니다.

### 접근 제어

```json5
{
  commands: {
    useAccessGroups: true,  // allowlist 기반 명령어 접근 제어 (기본: true)
  },
}
```

## 네이티브 명령어 vs 텍스트 명령어

| 항목       | 네이티브 명령어                   | 텍스트 명령어            |
| ---------- | --------------------------------- | ------------------------ |
| 실행       | 플랫폼 UI (슬래시, 버튼)         | 일반 채팅 메시지         |
| 세션       | 격리된 세션                       | 현재 채팅 세션           |
| 인수       | 자동완성/메뉴                     | 자유 텍스트              |
| 지원       | Discord, Telegram, Slack          | 모든 채널                |
