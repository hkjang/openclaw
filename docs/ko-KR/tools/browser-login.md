# 브라우저 로그인

브라우저 자동화 도구를 사용할 때 웹사이트에 로그인하는 방법과 X/Twitter 게시 워크플로우입니다.

## 핵심 원칙

> **수동 로그인을 권장합니다.** 자동화된 로그인은 봇 감지 시스템을 트리거할 수 있습니다.

OpenClaw는 `openclaw`이라는 이름의 **전용 Chrome 프로필**을 사용합니다 (주황색 UI 구분). 이 프로필은 일상 브라우저와 분리됩니다.

## 수동 로그인 (권장)

### 방법 1: 에이전트를 통해 열기

에이전트에게 브라우저를 열도록 요청한 뒤, 직접 로그인합니다:

1. 에이전트: "브라우저에서 X.com을 열어줘"
2. 브라우저가 열리면 직접 로그인
3. 로그인 완료 후 에이전트에게 작업 요청

### 방법 2: CLI로 열기

```bash
# 브라우저 시작
openclaw browser start

# 특정 URL로 열기
openclaw browser open https://x.com
```

### 방법 3: 특정 프로필 사용

```bash
openclaw browser open https://x.com --browser-profile openclaw
```

기본 프로필은 `openclaw`입니다.

## Chrome 프로필

### 전용 프로필

| 항목        | 설명                                  |
| ----------- | ------------------------------------- |
| **이름**    | `openclaw`                            |
| **구분**    | 주황색 UI로 일상 브라우저와 구분      |
| **쿠키**    | 일상 브라우저와 분리된 세션           |
| **목적**    | 에이전트 전용 자동화                  |

### 프로필 분리의 이점

- 일상 브라우저 세션에 영향 없음
- 자동화 전용 로그인 유지
- 봇 감지 리스크 감소

## X/Twitter: 권장 워크플로우

X/Twitter은 봇 감지가 엄격한 사이트입니다.

### 읽기/검색/스레드

**호스트 브라우저**에서 수동 로그인 후 사용:

1. 수동으로 X.com에 로그인
2. 에이전트에게 브라우저 조작 요청
3. 로그인된 세션으로 컨텐츠 읽기/검색

### 게시하기

**호스트 브라우저**에서 수동 로그인 후 게시:

1. 수동으로 X.com에 로그인
2. 에이전트가 트윗 작성 후 게시
3. 중요: 게시 전 내용 확인 권장

## 샌드박스와 호스트 브라우저 접근

### 샌드박스 세션의 제한

샌드박스에서 실행되는 브라우저 세션은 봇 감지를 **더 쉽게 트리거**합니다. X/Twitter같은 엄격한 사이트에서는 **호스트 브라우저**를 사용하세요.

### 호스트 브라우저 접근 허용

샌드박스 에이전트가 호스트 브라우저에 접근하도록 설정합니다:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,  // 호스트 브라우저 접근 허용
        },
      },
    },
  },
}
```

### CLI로 호스트 브라우저 열기

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

## 자동 로그인을 피해야 하는 이유

| 문제                     | 설명                                  |
| ------------------------ | ------------------------------------- |
| **봇 감지**              | 자동화된 로그인 패턴이 감지됨         |
| **CAPTCHA**              | 자동화 시 CAPTCHA가 더 자주 나타남    |
| **계정 잠금**            | 의심스러운 활동으로 계정 일시 정지    |
| **2FA 처리**             | 자동화로 2FA를 처리하기 어려움        |
| **자격 증명 노출**       | 에이전트에 비밀번호를 전달하면 보안 위험 |

## 로그인 유지 팁

1. **한 번 수동 로그인**: 전용 프로필에서 로그인하면 쿠키가 유지됩니다
2. **세션 만료 시**: 다시 수동으로 로그인하세요
3. **여러 사이트**: 필요한 모든 사이트에 미리 로그인해두세요
4. **프로필 백업**: Chrome 프로필 디렉토리를 백업할 수 있습니다

## 사이트별 참고사항

| 사이트        | 봇 감지 | 권장 방식                |
| ------------- | ------- | ------------------------ |
| **X/Twitter** | 엄격    | 호스트 브라우저, 수동 로그인 |
| **GitHub**    | 보통    | 전용 프로필에서 수동 로그인 |
| **Google**    | 엄격    | 전용 프로필에서 수동 로그인 |
| **LinkedIn**  | 엄격    | 호스트 브라우저, 수동 로그인 |
| **일반 사이트** | 낮음  | 전용 프로필 또는 자동화 가능 |
